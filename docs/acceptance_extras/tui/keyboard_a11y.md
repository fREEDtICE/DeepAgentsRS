## 1. 端到端效果

TUI 必须保证在纯键盘环境下可完成核心任务，并且不会出现“焦点丢失/无法退出/无法滚动”的致命问题。

端到端验收关注：

- 全流程无需鼠标
- 关键组件可聚焦（输入框、消息列表、工具卡片、审批对话框、线程列表）
- 基础快捷键可用且不冲突（至少：发送、取消、滚动、切换面板）

## 2. 验收方法

- 固定终端尺寸
- 用键盘脚本驱动（按键序列），产出屏幕快照
- 断言：焦点位置、可见内容、退出码

## 3. E2E 场景（必测）

### TKA-01：仅键盘完成一次对话

当：

- 聚焦输入框
- 输入文本并发送

则：

- 消息出现在消息列表

### TKA-02：滚动长消息/长工具结果

给定：

- 有超过一屏的内容

当：使用键盘滚动

则：

- 内容可滚动，焦点不丢失

### TKA-03：审批弹窗的键盘可操作性

给定：

- 出现审批弹窗

当：

- 使用 Tab/方向键切换选项
- Enter 确认

则：

- 审批生效，弹窗关闭

### TKA-04：可退出（不会卡死）

当：触发退出快捷键（例如 Ctrl+C 或 q，取决于实现）

则：

- TUI 正常退出，exit code 为 0（正常退出）或固定值（用户中断）
- 不留下悬挂子进程

## 4. 通过标准

- TKA-01 ~ TKA-04 全通过
